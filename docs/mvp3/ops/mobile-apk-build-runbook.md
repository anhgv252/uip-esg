# Mobile APK Build Runbook — UIP Operator App

**Status**: ✅ Ready for pilot distribution  
**Last Updated**: 2026-06-05  
**Owner**: DevOps Team

## Overview

This runbook documents the process to build Android APKs for the UIP Operator Mobile app using Expo EAS Build. The APK is distributed directly to pilot testers without Google Play Store submission.

## Prerequisites

### Required Tools
- **Node.js** 20+ (`node --version`)
- **EAS CLI** (`npm install -g eas-cli`)
- **Expo Account** with build credits or subscription
- **Android Keystore** (auto-generated by EAS on first build)

### Environment Setup
```bash
# Install EAS CLI globally
npm install -g eas-cli

# Login to Expo account
eas login

# Verify login
eas whoami
```

## First-Time Setup

### 1. Initialize EAS Project

```bash
cd applications/operator-mobile
eas init
```

This will:
- Create/link EAS project ID
- Update `app.json` with `extra.eas.projectId`

### 2. Configure Android Credentials

EAS will auto-generate a keystore on first build. To use a custom keystore:

```bash
eas credentials

# Select:
# → Android → Production
# → Keystore: Set up a new keystore
# → Upload your keystore.jks file
```

**Keystore Requirements**:
- **Format**: JKS or PKCS12
- **Alias**: Application package name (`com.uip.smartcity.operator`)
- **Validity**: 25+ years for production

### 3. Verify Build Configuration

Check `eas.json`:
```json
{
  "build": {
    "preview": {
      "distribution": "internal",
      "android": {
        "buildType": "apk",
        "gradleCommand": ":app:assembleRelease"
      }
    }
  }
}
```

## Building APK for Pilot Distribution

### Local Build (Manual)

```bash
cd applications/operator-mobile

# Build preview APK
npm run build:android:apk

# Or directly with EAS CLI
eas build --platform android --profile preview
```

**Build Process**:
1. EAS uploads project to cloud
2. Builds APK on EAS infrastructure
3. Returns download URL when complete
4. APK expires after 30 days

### CI/CD Build (Automated)

**Trigger**: Push to `main` branch with changes in `applications/operator-mobile/**`

**Manual Trigger**:
```bash
gh workflow run mobile-build.yml
```

**GitHub Secrets Required**:
- `EXPO_TOKEN`: Get from `eas whoami` → Settings → Access Tokens

### Check Build Status

```bash
# List recent builds
npm run build:check

# Or with EAS CLI
eas build:list --platform android --limit 5

# View specific build
eas build:view <build-id>
```

## Downloading & Installing APK

### Download from EAS Dashboard

1. Go to https://expo.dev/accounts/uip-smartcity/projects/operator-mobile/builds
2. Find the latest "preview" build
3. Click **Download APK**
4. Share download link with pilot testers

### Install on Test Device

**Method 1: QR Code (Recommended)**
```bash
# Generate QR code for testers
eas build:view <build-id>
```
Testers scan QR code → download APK → install

**Method 2: ADB Install**
```bash
# Download APK to local machine
curl -o operator-mobile.apk "<eas-download-url>"

# Connect Android device via USB
adb devices

# Install APK
adb install operator-mobile.apk

# Launch app
adb shell monkey -p com.uip.smartcity.operator -c android.intent.category.LAUNCHER 1
```

**Method 3: Direct Download on Device**
1. Open EAS build URL on Android device
2. Tap **Download**
3. Allow "Install from Unknown Sources" if prompted
4. Open APK → Install

## Troubleshooting

### Build Fails with "ENOENT: no such file or directory"

**Cause**: Missing `package-lock.json` or `node_modules`

**Fix**:
```bash
cd applications/operator-mobile
rm -rf node_modules package-lock.json
npm install
eas build --platform android --profile preview
```

### "Invalid keystore" Error

**Cause**: Keystore password mismatch or corrupted keystore

**Fix**:
```bash
# Reset credentials and generate new keystore
eas credentials
# Select: Android → Production → Keystore → Remove keystore → Generate new
```

### APK Install Fails on Device

**Cause**: Package name conflict with previous version

**Fix**:
```bash
# Uninstall old version first
adb uninstall com.uip.smartcity.operator

# Then install new APK
adb install operator-mobile.apk
```

### "Build Credits Exhausted"

**Cause**: Free tier allows limited builds/month

**Fix**:
- Upgrade to EAS Production plan ($29/month)
- Or wait for monthly quota reset

### CI/CD Build Hangs at "Waiting for build..."

**Cause**: `--no-wait` flag missing in workflow

**Fix**: Already configured in `.github/workflows/mobile-build.yml`

### API Base URL Not Updated in APK

**Cause**: `extra.apiBaseUrl` in `app.json` not read by app

**Fix**: Verify `Constants.expoConfig.extra.apiBaseUrl` in app code:
```typescript
import Constants from 'expo-constants';
const API_URL = Constants.expoConfig?.extra?.apiBaseUrl || 'http://localhost:3000';
```
