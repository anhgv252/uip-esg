## Live Secret Rotation — Day 3 (2026-06-20)

Requires live Keycloak at UAT environment. Steps are documented in the patch script. Summary:

```bash
KC_URL=http://localhost:8085  # (or UAT Keycloak URL)
KC_PASS=${KEYCLOAK_ADMIN_PASSWORD}

# 1. Get admin token
TOKEN=$(curl -sf -X POST "${KC_URL}/realms/master/protocol/openid-connect/token" \
  -d 'grant_type=password' -d 'client_id=admin-cli' \
  --data-urlencode "username=admin" --data-urlencode "password=${KC_PASS}" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['access_token'])")

# 2. Get uip-api internal ID
CLIENT_ID=$(curl -sf "${KC_URL}/admin/realms/uip/clients?clientId=uip-api" \
  -H "Authorization: Bearer $TOKEN" | python3 -c "import sys,json; print(json.load(sys.stdin)[0]['id'])")

# 3. Rotate secret (Keycloak generates cryptographically secure value)
NEW_SECRET=$(curl -sf -X POST "${KC_URL}/admin/realms/uip/clients/${CLIENT_ID}/client-secret" \
  -H "Authorization: Bearer $TOKEN" \
  | python3 -c "import sys,json; print(json.load(sys.stdin)['value'])")

# 4. Store and restart backend
echo "KEYCLOAK_CLIENT_SECRET=$NEW_SECRET" >> infrastructure/.env
docker compose up -d --no-deps backend
```

## Security Notes

- `uip-frontend` and `uip-mobile` are **public clients** (`publicClient: true`, `secret: ""`). Public clients do not use client secrets — PKCE provides the security guarantee.
- Only `uip-api` is a **confidential client** and requires secret rotation.
- After rotation, the old secret is immediately invalidated by Keycloak.
- Recommend storing the new secret in Vault / Kubernetes Secret, not plain `.env` for production.
